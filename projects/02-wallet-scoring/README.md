# Скоринг блокчейн-кошельков

## Что это

Сервис оценки активности кошельков Solana. На вход — поток транзакций из блокчейна. На выходе — числовой скор для каждого кошелька и категория для токенов (легитимный / подозрительный / скам). Идея в том, чтобы по поведенческим признакам отделить ботов и скам-проекты от реальных пользователей.

## Как устроено

Go Ingester слушает gRPC-стрим от Solana-ноды (endpoint задается в конфиге через `GRPC_ENDPOINT`, поднимать ноду или подключаться к стороннему провайдеру — задача того, кто разворачивает). Сырые события летят в Kafka, оттуда их забирает Go Writer и пишет в ClickHouse батчами. Python Scorer читает из ClickHouse, считает фичи (возраст кошелька, количество транзакций, число уникальных контрагентов, концентрация холдингов, средний размер транзакции), прогоняет через модель и пишет скоры обратно в отдельную таблицу. Go API на Gin отдает скоры наружу по HTTP.

## Стек

- Go 1.22, Gin, sarama (Kafka), clickhouse-go
- Python 3.11, XGBoost, PyTorch, pandas
- ClickHouse 24.3, Kafka 3.7 (KRaft), Redis 7
- Docker Compose

## ML-модель

Входные фичи:
- возраст кошелька (дни от первой транзакции)
- количество исходящих и входящих транзакций
- число уникальных контрагентов
- индекс концентрации холдингов (Herfindahl по балансам токенов)
- средний и медианный размер транзакции
- доля транзакций со взаимодействием со смарт-контрактами

Выход:
- скор кошелька от 0 до 100
- категория токена: `legit`, `suspicious`, `scam`

Алгоритм: XGBoost дает базовое предсказание на табличных фичах, поверх него идет FC-сеть на PyTorch (3 скрытых слоя), которая учится на остатках бустинга. Финальный скор — взвешенная комбинация.

## Запуск

```bash
docker-compose up -d
```

Переменные окружения:

| Переменная | Описание |
|------------|----------|
| `GRPC_ENDPOINT` | gRPC-эндпоинт Solana-ноды |
| `GRPC_TOKEN` | Токен авторизации (если требуется провайдером) |
| `CLICKHOUSE_DSN` | DSN ClickHouse, например `clickhouse://default@clickhouse:9000/wallets` |
| `KAFKA_BROKERS` | `kafka:9092` |
| `REDIS_ADDR` | `redis:6379` |
| `API_PORT` | Порт для Go API, по умолчанию `8080` |

## Структура проекта

```
.
├── cmd/
│   ├── ingester/        # gRPC-клиент Solana, шлет в Kafka
│   ├── writer/          # читает Kafka, пишет в ClickHouse
│   └── scorer-api/      # HTTP API на Gin
├── internal/
│   ├── kafka/           # обертки над sarama
│   ├── clickhouse/      # клиент и репозитории
│   ├── solana/          # парсинг транзакций
│   └── scoring/         # бизнес-логика скоринга
├── ml/
│   ├── train.py         # обучение модели
│   ├── predict.py       # инференс, пишет скоры в ClickHouse
│   └── model/           # сохраненные веса
├── migrations/          # SQL для ClickHouse
├── docker-compose.yml
└── README.md
```
