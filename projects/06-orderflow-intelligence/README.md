# Анализ потока ордеров на DEX

На децентрализованных биржах каждый обмен токенов — публичная транзакция в блокчейне. Поток этих транзакций можно анализировать так же, как биржевой стакан на Мосбирже или NYSE: видно, кто покупает, кто продает, какими объемами и в каком темпе.

## Что это

Сервис разбирает поток свопов на DEX Solana (Raydium, Orca, Jupiter) и на основе агрегированных метрик предсказывает краткосрочное направление цены пары. Это аналог order flow analysis из традиционных финансов, адаптированный под особенности крипто-бирж: вместо стакана с лимитными заявками здесь поток исполненных сделок против пула ликвидности.

## Как устроено

Go Parser слушает Geyser gRPC (endpoint задается в конфиге через `GRPC_ENDPOINT`). Парсит swap-инструкции для трех DEX вручную: декодит сырые байты по известным layout'ам Raydium, Orca и Jupiter без зависимости от сторонних SDK. На выходе — нормализованное событие свопа: пара, направление, объем в обоих токенах, цена, slot, signature.

События улетают в Kafka. Go Feature Engine собирает их в окна 1m/5m/15m и для каждого окна по каждой паре считает OFI, VPIN, price impact, volume profile. Готовые фичи пишутся в ClickHouse.

Python ML читает фичи из ClickHouse и крутит модель: XGBoost на табличных признаках за окно плюс LSTM на последовательности окон. Предсказания возвращаются в ClickHouse. Go API на Gin отдает их по HTTP вместе с сырыми метриками.

## Метрики

**OFI (Order Flow Imbalance).** Разница между объемом покупок и продаж за период. Считается на агрегате свопов: если за минуту купили на 100 SOL, а продали на 30 SOL, OFI равен +70. Сильно положительный OFI означает давление покупателей, сильно отрицательный — давление продавцов.

**VPIN (Volume-Synchronized Probability of Informed Trading).** Оценка доли «информированных» трейдеров в общем потоке. Считается по бакетам равного объема, а не равного времени. Высокий VPIN означает, что сделки идут с явным перекосом в одну сторону: кто-то знает что-то, чего пока не знает рынок, и под это знание выгребает ликвидность.

**Price impact.** Сдвиг цены пары за окно, нормированный на объем. Показывает, насколько «дорогим» был поток для рынка.

**Volume profile.** Распределение объема по ценовым уровням внутри окна. Видны зоны интереса.

## ML

Таргет — направление цены через 5 минут после конца окна: `up`, `down`, `flat`. Классы разделяются по порогу процентного изменения (например, ±0.3%).

Фичи на вход:
- OFI за окно и его динамика (delta к предыдущему окну)
- VPIN
- price impact
- средний размер свопа
- доля свопов в одну сторону
- кумулятивный объем
- ширина диапазона цены внутри окна

Архитектура: XGBoost дает предсказание на табличных фичах одного окна. Параллельно LSTM (PyTorch, 2 слоя по 64 нейрона) смотрит на последовательность из 12 окон и ловит более длинные паттерны. Итоговая вероятность — взвешенное среднее двух выходов, веса подбираются на валидации.

## Запуск

```bash
docker-compose up -d
```

Переменные:

| Переменная | Описание |
|------------|----------|
| `GRPC_ENDPOINT` | Geyser gRPC-эндпоинт Solana-ноды |
| `GRPC_TOKEN` | Токен авторизации |
| `KAFKA_BROKERS` | `kafka:9092` |
| `CLICKHOUSE_DSN` | DSN ClickHouse |
| `REDIS_ADDR` | `redis:6379` |
| `WATCHED_PAIRS` | список пар для отслеживания через запятую |
| `API_PORT` | по умолчанию `8080` |
