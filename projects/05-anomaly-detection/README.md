# Детектор аномалий в транзакциях

## Что это

Real-time детекция подозрительных финансовых транзакций. На вход — поток в Kafka. На выходе — алерты по транзакциям, которые выбиваются из поведения клиента. Цель в том, чтобы поймать аномалию за секунды.

## Как устроено

В Kafka сыпется входной топик с транзакциями. Go Feature Extractor вытаскивает каждое сообщение и для конкретной транзакции считает скользящие фичи через Redis:

- средний объем операций клиента за последний час и за 24 часа (ZSET с timestamp в качестве score)
- количество уникальных контрагентов за 24 часа (HyperLogLog)
- z-score суммы транзакции относительно истории клиента
- время с предыдущей транзакции
- доля транзакций в нерабочие часы

Готовые фичи летят в отдельный Kafka-топик. Python ML Worker читает их и прогоняет через два уровня. Сначала Isolation Forest как быстрый скрининг: подавляющее большинство транзакций отсеивается дешево. Подозрительные доезжают до Autoencoder на PyTorch, где anomaly score считается как reconstruction error.

Результат пишется в ClickHouse (исторический лог) и одновременно публикуется в Redis pub/sub для подписчиков-алертеров. Go API на Gin отдает последние алерты, статистику по интервалам и состояние модели.

## ML

**Isolation Forest.** Учится в офлайне на исторических транзакциях, дообучается раз в сутки. Дешево вычисляется на инференсе, отсекает явный мусор. Контаминация выставлена в 0.01.

**Autoencoder.** PyTorch, простая архитектура: вход 12 фичей, энкодер `12 → 8 → 4`, декодер `4 → 8 → 12`. Лосс — MSE. На инференсе считаем reconstruction error, переводим в anomaly score через перцентильную нормализацию по последним 24 часам. Порог срабатывания — 99-й перцентиль, настраивается в конфиге.

Финальный алерт выпускается, если обе модели согласны: Isolation Forest вернул `-1` и autoencoder выдал score выше порога.

## Запуск

```bash
docker-compose up -d
```

Переменные:

| Переменная | Описание |
|------------|----------|
| `KAFKA_BROKERS` | `kafka:9092` |
| `KAFKA_INPUT_TOPIC` | топик с сырыми транзакциями |
| `KAFKA_FEATURES_TOPIC` | топик с готовыми фичами |
| `CLICKHOUSE_DSN` | DSN для логирования алертов |
| `REDIS_ADDR` | `redis:6379` |
| `ANOMALY_THRESHOLD` | порог срабатывания, по умолчанию `0.99` |
| `API_PORT` | по умолчанию `8080` |
