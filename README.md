## Общие принципы

Все проекты развертываются в контейнеризированной среде с использованием Docker Compose. Каждый проект содержит полный набор необходимых сервисов и может быть запущен локально без дополнительной настройки инфраструктуры.

Go-компоненты организованы по принципам Clean Architecture: точки входа расположены в каталоге `cmd`, внутренняя бизнес-логика — в `internal`. Для сборки используются многоэтапные Docker-образы, что позволяет минимизировать размер итоговых контейнеров.

ML-компоненты реализованы на Python и выделены в отдельные сервисы. Такой подход обеспечивает независимое масштабирование вычислительных задач и упрощает интеграцию моделей машинного обучения в общую систему.

Для тестирования и демонстрации предусмотрены встроенные инструменты генерации данных, позволяющие воспроизводить рабочие сценарии без необходимости использования производственных наборов данных.
